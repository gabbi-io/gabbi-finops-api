vPoC-FinOps-1.0 • HTML de storytelling (apresentação)
FinOps (Fase 1) — “Custo auditável” com saída pronta para Grafana
Objetivo: demonstrar ao cliente como cada requisito é atendido, conectando tela + narrativa + evidência.
Conceito central: Interaction → Accumulator → CostLedger → Grafana
📌 Ver mapa de requisitos
🎙️ Roteiro executivo (5 minutos)
Fala pronta, com “passo de tela” para ancorar o cliente.
DEMO
1) “Aqui a ideia é simples: transformar custo de IA em algo financeiro e auditável.”
“Custo não pode ser sensação — precisa virar ledger, com rastreabilidade e mínimos de billing.”
2) “Tudo começa nas Interactions: registramos cada uso, sem caixa preta.”
“Isso é a prova de auditoria do consumo (tokens, modelo, origem manual/automação).”
3) “Depois vem o balde (Accumulator): acumulamos tokens até fechar o mínimo.”
“Isso reproduz o comportamento real de cobrança mínima: fecha quando atinge min_tokens.”
4) “Quando fecha, geramos o CostLedger: isso é o que vai para o Grafana.”
“Ou seja: custo vira linha financeira, pronta para métricas e dashboard.”
5) “E o Grafana entra aqui: o layout já está preparado para embutir o painel real.”
“Quando estiver pronto, é só colar a URL do embed — UX não muda.”
Sequência de telas
Clique e siga na hora.
  1. Dashboard: mostrar KPIs + gráficos
  2. No Dashboard: Fake Live → “Reset Live” → “Simular 1 hora”
  3. Interactions: provar auditabilidade
  4. Accumulators: mostrar “baldes” e sobras
  5. Ledger: mostrar linhas financeiras
  6. Pricing: alterar min_tokens e repetir simulação
  7. Settings: mostrar slot do Grafana iframe
Dica: finalize com “Explodir custo” para storytelling e “Top Tasks/Flows” para governança.
Fechamento (1 frase)
Para encerrar bonito.
“A Fase 1 entrega custo auditável e pronto para Grafana; na Fase 2 entramos com receita/ROI e na Fase 3 rateio por área.”
🧠 Roteiro técnico (10–12 minutos)
Fala pronta para explicar arquitetura/funcionamento e “por que isso vira Grafana”.
FASE 1
1) Conceito: por que “custo” não é o token — é o ledger fundamento
Fala sugerida:
“Tokens são consumo técnico. O que o negócio precisa é custo financeiro consolidado. Por isso a Fase 1 termina em CostLedger: o ledger é o dado mestre do custo e vira Grafana.”
  • Abrir Ledger e mostrar colunas (amount, tokens_billed, source_type).
  • Voltar no Dashboard e mostrar “Custo total / Manual / Automação”.
2) Auditabilidade: Interaction como evidência controle
Fala sugerida:
“Aqui a gente tira a sensação de caixa preta: toda chamada gera uma Interaction. Mesmo antes de virar custo, já existe rastreio por origem, modelo e contexto.”
  • Abrir Interactions.
  • Mostrar campos: origem manual/automação, modelo, tokens, conversation_id.
  • Conectar: “o ledger é derivado disso, com mínimos e fechamento”.
3) Mínimo de billing: Accumulator (baldes) e fechamento billing
Fala sugerida:
“O ‘segredo’ do custo real é o mínimo. Então nós acumulamos tokens por bucket até fechar o mínimo (min_tokens) e então geramos o ledger. Isso replica o mundo real.”
  • Abrir Accumulators.
  • Mostrar pending_tokens e close_count.
  • Voltar no Dashboard: clicar “Simular 1 hora” e ver close_count mudar.
4) Pricing controlável: mudanças afetam o ledger governança
Fala sugerida:
“Pricing é governança. Se eu ajusto min_tokens/min_cost, o comportamento do fechamento muda. Isso permite simular cenários, aprovar regras e depois só trocar a fonte do dado.”
  • Abrir Model Pricing.
  • Alterar um modelo (ex.: min_tokens e custo).
  • Voltar ao Dashboard → “Reset Live” → “Simular 1 hora”.
5) Storytelling: crescimento diário e explosão de custo demo
Fala sugerida:
“Agora eu provo a necessidade do FinOps: quando há crescimento ou um spike, o painel mostra onde está o custo e em qual automação (task/flow).”
  • No Dashboard: selecionar cenário “Explosão”.
  • Clicar “Explodir custo”.
  • Mostrar “Top Tasks/Flows”.
  • Abrir Ledger e mostrar linhas novas.
6) Saída Grafana: slot de embed pronto observabilidade
Fala sugerida:
“A interface já nasce preparada para o Grafana real: quando os dados reais estiverem coletados, embutimos o dashboard sem refazer UX.”
  • Abrir Settings.
  • Mostrar campo “Grafana embed URL”.
  • Voltar no Dashboard e mostrar o card Grafana (slot).
